  <name>Task 2: Convert schema to migrations and add the RLS-fix migration</name>
  <files>supabase/migrations/, supabase/schema.sql</files>
  <action>
    Create two timestamped migrations using the CLI so they get canonical timestamps (do not hand-name timestamps):
    1. `npx supabase migration new initial_schema` — paste the FULL current contents of `supabase/schema.sql` into the generated file. This becomes the reproducible source of the existing 11-table schema + triggers.
    2. `npx supabase migration new rls_fixes` — add the security + performance fixes from research (Schema RLS Audit + Code Examples):
       - `DROP POLICY IF EXISTS "Allow authenticated users to insert/update prices" ON public.price_cache;`
       - `DROP POLICY IF EXISTS "Allow authenticated users to insert news" ON public.news_items;`
       (The existing SELECT-only policies on both tables remain, so authenticated users keep read access; writes are now reserved for the service role, which bypasses RLS.)
       - Add indexes on the RLS join columns:
         `CREATE INDEX IF NOT EXISTS idx_investment_accounts_user_id ON public.investment_accounts(user_id);`
         `CREATE INDEX IF NOT EXISTS idx_holdings_account_id ON public.holdings(account_id);`
         `CREATE INDEX IF NOT EXISTS idx_watchlist_items_account_id ON public.watchlist_items(account_id);`
         `CREATE INDEX IF NOT EXISTS idx_alerts_account_id ON public.alerts(account_id);`
         `CREATE INDEX IF NOT EXISTS idx_brokers_account_id ON public.brokers(account_id);`
    Keep `supabase/schema.sql` in place as the human-readable reference (add a one-line comment at its top noting the canonical source is now `supabase/migrations/`). Do NOT delete it.
    Do not rewrite the already-correct per-user table policies; only the two shared-table write policies are holes.
  </action>
  <verify>`ls supabase/migrations/*_initial_schema.sql supabase/migrations/*_rls_fixes.sql` lists both files; `grep -rn "DROP POLICY" supabase/migrations/*_rls_fixes.sql` shows the two drops.</verify>
