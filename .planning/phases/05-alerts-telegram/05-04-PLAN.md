2. `dispatchOutbox(admin): Promise<DispatchSummary>` — the sole Telegram-sending path (05-RESEARCH-schema-outbox "Dispatch Architecture"):
   a. `const { data: claimed } = await admin.rpc('claim_due_notifications', { p_limit: 25 });` (bounded batch for function-duration hygiene). If none, return zeroed summary.
   b. Batch-resolve recipients: `admin.from('telegram_links').select('user_id, chat_id, status').in('user_id', userIds)`; build a map of `user_id → chat_id` for rows where `status === 'linked' && chat_id !== null`.
   c. Iterate claimed rows SEQUENTIALLY (per-chat, never Promise.all — 05-RESEARCH-telegram-api Q4 rate-limit compliance). For each row:
      - No linked chat → `.update({ status:'failed', last_error:'no linked telegram chat' })` (honest, do not retry what can't succeed); summary.failed++.
      - Else `const r = await sendTelegramMessage(chatId, row.payload.text);`
        - `r.ok` → `.update({ status:'sent', sent_at: nowIso, last_error: null })`; summary.sent++.
        - else `const c = classifySendError(r.errorCode, r.description, r.retryAfter);`
          - `c.kind === 'retryable'` → leave status='pending'; if 429 with `c.retryAfterSeconds`, overwrite `next_attempt_at = now + retryAfterSeconds*1000` (honest, API-provided); record `last_error = r.description`; summary.retried++. (The claim already bumped attempts+backoff; a 429 refines it with the real wait.)
          - `c.kind === 'permanent'` → `.update({ status:'failed', last_error: r.description })`; summary.failed++. If the description indicates blocked/chat-not-found, ALSO flip that user's `telegram_links` row to `status='revoked'` (keeps the allowlist honest — 05-RESEARCH-schema-outbox Dispatch; user must re-handshake).
   d. Return the DispatchSummary. This function NEVER throws for a single bad send — one poisoned row does not abort the batch (fetchPrices precedent).

```ts
import { NextResponse } from 'next/server';
import { isAuthorizedRefreshRequest } from '@/lib/prices/ingest';
import { dispatchOutbox } from '@/lib/notifications/outbox';
import { createAdminClient } from '@/utils/supabase/admin';

// ALRT-05 — standalone secret-guarded dispatch lever: the manual "a message
// failed, send now without waiting for the next price refresh" path, the local
// verification path (05-09), and the future deploy-time cron target. Reuses the
// SAME pure predicate as /api/prices/refresh: an empty/unset secret ALWAYS
// denies. Guard runs BEFORE createAdminClient() — an unauthorized request never
// touches Supabase.
export async function POST(request: Request) {
  const authHeader = request.headers.get('authorization');
  if (!isAuthorizedRefreshRequest(authHeader, process.env.NOTIFY_DISPATCH_SECRET ?? '')) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
  }
  try {
    const summary = await dispatchOutbox(createAdminClient());
    return NextResponse.json({ success: true, ...summary });
  } catch (error) {
    console.error('Outbox dispatch failed:', error);
    return NextResponse.json(
      { error: error instanceof Error ? error.message : 'Unknown error' },
      { status: 500 }
    );
  }
}
```
