  <action>
Replace the localStorage-backed implementation with one backed by `public.yt_channels` (columns from the Phase 1 schema: `channel_id`, `account_id`, `channel_name`, `is_active`, `created_at` — composite PK `(channel_id, account_id)`).

Design the hook's public API as explicit async operations instead of a generic `setChannels` state-setter (the old API assumed synchronous localStorage writes; DB writes are async and can fail):

```typescript
export function useChannels() {
  // returns: { channels: YTChannel[], isLoaded: boolean, error: string | null,
  //            addChannel: (ch: Omit<YTChannel,'is_active'> & {is_active?: boolean}) => Promise<void>,
  //            toggleChannel: (channelId: string) => Promise<void>,
  //            removeChannel: (channelId: string) => Promise<void> }
}
```

Implementation notes:
- Resolve the current user's `account_id` via `supabase.from('investment_accounts').select('id').eq('user_id', user.id).single()` (small inline helper — do not import from any Phase 2 portfolio data-layer file; this plan is intentionally independent of those, some duplication of this ~3-line lookup is fine and documented as such in a comment).
- On mount: fetch `supabase.from('yt_channels').select('*').eq('account_id', accountId)`. Map DB rows to the existing `YTChannel` shape from `src/lib/mock-youtube-data.ts` (keep using that type — it is NOT part of the mock-portfolio deletion scope; only the channel list SOURCE changes, not the type or the video-analysis mock data).
- If the fetch returns zero rows, set `channels: []` — do NOT fall back to `MOCK_CHANNELS`. This is a deliberate behavior change from the old hook (which seeded MOCK_CHANNELS on empty); the cross-cutting project rule is "never silently fall back to mock data," and an empty tracked-channel list is a legitimate, honest state.
- `addChannel`: `supabase.from('yt_channels').insert({ channel_id, account_id, channel_name, is_active: true })`, then refetch or optimistically append on success.
- `toggleChannel`: `supabase.from('yt_channels').update({ is_active: !current }).eq('channel_id', id).eq('account_id', accountId)`.
- `removeChannel`: `supabase.from('yt_channels').delete().eq('channel_id', id).eq('account_id', accountId)`.
- Surface failures via a returned `error: string | null` rather than swallowing them silently (the old hook only `console.error`'d localStorage failures).
- `avatar_color` and `subscriber_count`/`video_count` fields on `YTChannel` (used for UI display, not in the `yt_channels` table) — since these are UI-cosmetic and not persisted columns in the Phase 1 schema, keep them client-computed (e.g. `avatar_color` derived deterministically from `channel_name`, `subscriber_count`/`video_count` defaulted to values already returned by the `/api/youtube/channel` resolve call at add-time and not persisted/refetched). Document this as an intentional simplification, not a bug.
