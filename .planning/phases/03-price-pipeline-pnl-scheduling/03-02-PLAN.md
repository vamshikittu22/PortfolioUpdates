**`src/lib/prices/ingest.ts`** — functions used by the refresh cycle (consumed by 03-03's network wrappers and 03-04's orchestration/route):

`parseYahooChartResponse(json: unknown): { price: number; previousClose: number; changePct: number } | null`
- Input shape matches Yahoo's `/v8/finance/chart/{symbol}` response: `json.chart.result[0].indicators.quote[0].close` is an array of numbers (may contain `null` for non-trading days).
- Filter out `null`/`undefined` entries from `close`. If zero valid entries remain, or the expected path (`chart.result[0]`) is missing/malformed in any way, return `null` — NEVER throw, never return a guessed number.
- `price` = last valid close. `previousClose` = second-to-last valid close if at least two valid entries exist; otherwise `previousClose = price` and `changePct = 0` (a single data point has no day-change to report — this is not the same as failure, so still returns a value, just with 0 change).
- `changePct = ((price - previousClose) / previousClose) * 100`, guarding division by zero (`previousClose === 0` → `changePct = 0`).

`detectCorporateAction(changePct: number): boolean`
- Returns `true` when `Math.abs(changePct) > 40`, else `false`. Strict `>`, so exactly 40 is `false` (documented boundary — matches the roadmap's ">40%" wording literally).

`shouldSkipRefresh(lastUpdatedAt: Date | null, now: Date, minIntervalMs = 60_000): boolean`
- `lastUpdatedAt === null` → `false` (never fetched, must fetch — this is the "no fabricated first value" path feeding into 03-04).
- `now.getTime() - lastUpdatedAt.getTime() < minIntervalMs` → `true` (too recent, skip — this is the thundering-herd/dedup guard from 03-RESEARCH.md Pitfall 1).
- Otherwise → `false`.

`isAuthorizedRefreshRequest(authHeader: string | null, expectedSecret: string): boolean`
- Returns `true` only when `authHeader === \`Bearer ${expectedSecret}\`` AND `expectedSecret` is a non-empty string. An empty/unset `expectedSecret` (e.g. env var missing) must ALWAYS return `false`, even if `authHeader` is also empty or `'Bearer '` — this prevents an unconfigured secret from silently becoming an open endpoint. This is the pure predicate 03-04's route handler calls before touching Supabase.

**`src/lib/prices/pnl-calculator.ts`** — functions used by the P&L display (consumed by 03-05):

`convertToBaseCurrency(amount: number, fromCurrency: 'INR' | 'USD', baseCurrency: 'INR' | 'USD', fxRate: number): number`
- `fromCurrency === baseCurrency` → return `amount` unchanged, regardless of `fxRate` (identity case — do not silently multiply same-currency amounts).
- Otherwise → `amount * fxRate`, where `fxRate` is the caller-supplied rate already oriented `fromCurrency → baseCurrency` (the caller, 03-05, is responsible for picking the correctly-oriented cached rate; this function does no lookup itself — stays pure).
- Sign is preserved (a loss converts to a loss, not `abs()`'d).

`calculateHoldingPnL(holding: { quantity: number; avgCost: number; currency: 'INR' | 'USD' }, currentPrice: number | null, changePct: number | null): HoldingPnL`
- `currentPrice === null` → return a `{ status: 'pending', costBasis: quantity * avgCost, currentValue: null, unrealizedPnL: null, unrealizedPnLPct: null, dayChangeAmount: null, dayChangePct: null }` shape — the "honest pending state" PRICE-04 requires. Never substitute `0` for `currentValue`/`unrealizedPnL` here.
- `currentPrice` present → `status: 'priced'`, `costBasis = quantity * avgCost`, `currentValue = quantity * currentPrice`, `unrealizedPnL = currentValue - costBasis`, `unrealizedPnLPct = costBasis > 0 ? (unrealizedPnL / costBasis) * 100 : 0` (guard divide-by-zero for a zero-cost-basis edge case), `dayChangeAmount = currentValue * ((changePct ?? 0) / 100)`, `dayChangePct = changePct ?? 0`.

`calculatePortfolioTotals(holdingsPnL: Array<HoldingPnL & { currency: 'INR' | 'USD' }>, baseCurrency: 'INR' | 'USD', fxRate: number): PortfolioTotal`
- Sums `costBasis` and (for `status: 'priced'` holdings only — `pending` holdings contribute their cost basis but `null` current value) `currentValue`/`unrealizedPnL`/`dayChangeAmount`, converting each non-base-currency holding via `convertToBaseCurrency` BEFORE summing.
- Returns both the converted total (`totalCostBasis`, `totalCurrentValue`, `totalUnrealizedPnL`, `totalDayChange`, all in `baseCurrency`) AND a `nativeSubtotals: Record<'INR'|'USD', { costBasis: number; currentValue: number | null }>` breakdown — this is what makes the FX effect VISIBLE per PRICE-06 (a caller can show "₹50,000 (native) + $1,000 (native) = ₹1,33,000 (@ 83.5 USD→INR)" instead of one opaque number).
- `fxRateUsed: fxRate` is echoed back in the result so the UI can display it directly.
- A portfolio with zero holdings, or all holdings `pending`, must not throw or produce `NaN` — totals default to `0`/`null` cleanly.
