  <action>
Define:

```typescript
export interface PriceFetchResult {
  price: number;
  changePct: number;
  fetchError: null;
} | {
  price: null;
  changePct: null;
  fetchError: string;
}

export async function fetchPrices(symbols: string[]): Promise<Record<string, PriceFetchResult>>
```

For each `symbol` in `symbols` (these are ALWAYS `instruments.price_source_symbol` values, e.g. `'INFY.NS'`, `'TATASTEEL.BO'`, `'AAPL'`, `'INFY'` for the NYSE ADR — never the bare display `symbol` column; per 03-RESEARCH.md Pitfall 3, mixing the two would fetch the wrong exchange's quote):
1. `GET https://query2.finance.yahoo.com/v8/finance/chart/${symbol}?interval=1d&range=5d` with the same `User-Agent` header used in `src/lib/research/yahoo-finance.ts` (Yahoo blocks requests with no browser-like UA).
2. On non-OK HTTP status or a thrown network error: `results[symbol] = { price: null, changePct: null, fetchError: <message, e.g. \`HTTP ${res.status}\` or the caught error's message> }`. Do NOT let one symbol's failure abort the loop — wrap each iteration's body in try/catch (or use `Promise.allSettled` across all symbols for parallelism; either is acceptable, prefer `Promise.allSettled` for speed on a 15–20 symbol portfolio, but make sure a single rejected promise still produces a per-symbol error result, not an unhandled rejection).
3. On success: `await res.json()`, pass to `parseYahooChartResponse` from `@/lib/prices/ingest`. If it returns `null` (malformed shape), treat exactly like a fetch failure: `{ price: null, changePct: null, fetchError: 'Malformed response from price source' }`. If it returns a value, map to `{ price: result.price, changePct: result.changePct, fetchError: null }`.

This function does NOT know about `instrument_id`, Supabase, or the cache — it is keyed purely by the symbol string it was asked to fetch. Mapping symbol → instrument_id happens in the caller (plan 03-04).
