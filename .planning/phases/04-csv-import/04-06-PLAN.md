  <name>Task 2: ImportPage client container (progressive state machine)</name>
  <files>src/components/import/ImportPage.tsx</files>
  <action>
`'use client'`. This is the single owner of import interaction state. Implement a progressive (NOT wizard) flow — everything visible at once, sections revealed as data arrives:

State: `phase: 'idle' | 'previewing' | 'preview' | 'committing' | 'done'`, `file: File | null`, `preview: (ImportPreview & {ok:true}) | null`, `previewError: string | null`, `choices: CommitChoices` (default `{ importDuplicates: false, importUnsupported: false, mappings: [] }`), `result` / `commitError`.

Flow:
1. Render `<ImportDropzone onFile={handleFile} disabled={phase==='previewing'||phase==='committing'} />` at top always (until `done`).
2. `handleFile(file)`: store it, set `phase:'previewing'`, build `FormData` with `file`, call `previewImport(fd)` inside `startTransition`. On `{ok:true}` → store preview, `phase:'preview'`. On `{ok:false}` → show `previewError` (honest server message, incl. ImportParseError text). Optionally expose the manual broker override: if the error is broker-detection, show a Groww/Robinhood picker that re-calls previewImport with a `broker` field added to the FormData.
3. When `phase==='preview'`: render, below the dropzone, in order — the prior-batch banner (if `preview.priorBatch`, "This exact file was imported on DATE — N imported, M duplicates"), then `<SymbolMappingSection>` (only if `preview.unmatchedSymbols.length`), then `<PreviewTable preview choices onChoicesChange={setChoices} />`, then a commit bar at the bottom.
4. Commit button: disabled while any unmatched symbol the user intends to import lacks a mapping in `choices.mappings` (compute from `preview.unmatchedSymbols` minus resolved) — enforces "nothing silently dropped". On click: `phase:'committing'`, build `FormData` with the SAME `file` and `choices: JSON.stringify(choices)`, call `commitImport(fd)`. On `{ok:true}` → `result`, `phase:'done'`. On `{ok:false}` → `commitError`, back to `phase:'preview'` (user can retry).
5. When `phase==='done'`: replace the preview area with `<ImportSummary result={result} onReset={reset} />` (in place — no navigation). `reset()` returns to `idle` with cleared state.

Surface every error inline (never swallow), matching the isPending/error idioms in RefreshPricesButton/HoldingFormDialog. Use `useTransition` for both Server Action calls.
  </action>
