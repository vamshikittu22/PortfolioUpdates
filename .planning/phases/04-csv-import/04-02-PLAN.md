**`src/lib/import/normalize.ts`** тАФ every function returns `null` on failure; NEVER `0`, never a guess.

`parseMoney(raw: string | null): number | null`
- `null`, `''`, `'тАФ'`, `'-'` тЖТ `null` (blank/placeholder, e.g. a Robinhood dividend row's Price). Parentheses mean negative (`($43.64)` тЖТ `-43.64`); strip a leading `$` and thousands commas; if the cleaned remainder is not `^\d+(\.\d+)?$` тЖТ `null` (caller marks the ROW invalid, never coerces to 0). Sign preserved.

`parseQuantity(raw: string | null): number | null`
- Trim; strip thousands commas; `''`/`null` тЖТ `null`. Must accept fractional shares to тЙе6 dp (`0.123456`). Non-numeric тЖТ `null`. Zero or negative тЖТ `null` (Phase 2's transactions CHECK is `quantity > 0`; a 0/negative quantity is a row validation failure, not a silent import).

`parseRobinhoodDate(raw: string | null): string | null`
- `^(\d{1,2})\/(\d{1,2})\/(\d{4})$` (M/D/YYYY) тЖТ `YYYY-MM-DD` with zero-padding. Anything else (including a trailing disclaimer sentence) тЖТ `null`.

`parseGrowwDate(raw: string | null): string | null`
- Accept `DD-MM-YYYY`, `DD/MM/YYYY`, and `DD MMM YYYY` (e.g. `18 Sep 2023`) тЖТ ISO `YYYY-MM-DD`. Unrecognized тЖТ `null` (caller falls back to today with a visible note тАФ that fallback lives in the parser, 04-03, not here).

**`src/lib/import/row-hash.ts`**

`computeRowHashes(rows: NormalizedRow[]): string[]`
- For each row build `base = [broker, isin ?? symbol, txnType, quantityStr, priceStr ?? '', dateISO].join('|')`. Maintain a `Map<string, number>` of occurrence counts; the 1-based occurrence index `n` for this identical `base` is appended: `sha256(\`${base}|${n}\`)` via `node:crypto` `createHash('sha256').digest('hex')`. Returns hashes positionally aligned to `rows`. Purity: no `Date.now()`, no randomness тАФ same input array yields identical output every call.

**`src/lib/import/detect-broker.ts`**

`detectBroker(bytes: Uint8Array, fileName: string): 'groww' | 'robinhood' | 'unknown'`
- Zip magic (`bytes[0]===0x50 && bytes[1]===0x4b`, i.e. `PK`) тЖТ `'groww'` (every .xlsx is a zip; the header-scan parse confirms it). Else decode the first ~2048 bytes as text: if it matches `/Trans Code/i` AND `/Activity Date/i` тЖТ `'robinhood'`. Else `'unknown'`. `fileName` is available as a weak tiebreaker (`.xlsx`/`.csv`) but MUST NOT override the byte evidence.
