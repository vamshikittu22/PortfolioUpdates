  <behavior>

**`src/lib/import/parse-robinhood.ts`** — `parseRobinhood(text: string): ParsedRow[]`
- `Papa.parse(text, { header: true, skipEmptyLines: true })`. Access columns BY NAME (`row['Trans Code']`, `row['Instrument']`, `row['Quantity']`, `row['Price']`, `row['Activity Date']`) — handles the 9- and 11-column variants. Never positional.
- Map `Trans Code`: `Buy`→`BUY`, `Sell`→`SELL`, `SPL`→`SPLIT` (price null, quantity = shares received). Everything else (`CDIV`, `SPR`, `CONV`, `SXCH`, `MRGS`, `OEXP`, `OASGN`, `ACH`, `INT`, `DFEE`, `GOLD`, `MRGN`, unknown) → `status: 'unsupported'` with `statusReason` naming the code (reported, not dropped).
- Use `parseRobinhoodDate`, `parseMoney` (Price), `parseQuantity` (Quantity) from 04-02. A row whose date fails to parse (footer/disclaimer) is skipped entirely (not emitted). A supported row where quantity/price normalization fails → `status: 'invalid'` with a reason (never coerced to 0). BUY/SELL require a numeric price; SPLIT has null price.
- `isin: null` for every Robinhood row (no ISIN in the file). `symbol = row['Instrument']`.

**`src/lib/import/parse-groww.ts`** — `parseGroww(bytes: Uint8Array): ParsedRow[]`
- `XLSX.read(bytes, { type: 'array' })`; take the first sheet; `XLSX.utils.sheet_to_json(ws, { header: 1, raw: false, defval: null })` (array-of-arrays).
- **Header scan:** find the first row (within the first ~30) that has a cell normalizing to `isin` AND a cell matching `/quant/i`. If none → `throw new ImportParseError('Unrecognized Groww format: no header row containing "ISIN" ... First rows seen: ' + JSON.stringify(grid.slice(0,5)))`. Build a name→index map from that header row (trim, lowercase, collapse spaces). Match columns fuzzily: `isin`; quantity via `/quant/`; average price via `/(average|avg).*price/` or `/buy price/`; stock/display name via `/(stock|name|company)/`.
- Read data rows AFTER the header until the first row whose ISIN cell fails `^[A-Z]{2}[A-Z0-9]{9}[0-9]$` (drops totals/footer naturally). A valid-ISIN row whose prefix is `INF` (mutual fund) → `status: 'unsupported'` (reported; equity filtering also happens at match time). 
- **Snapshot → synthetic BUY:** each equity row becomes ONE `ParsedRow` with `txnType: 'BUY'`, `quantity` = parsed Quantity, `price` = parsed Average buy price, `dateISO` = statement date (scan the title block above the header for a `DD-MM-YYYY`/`DD MMM YYYY` via `parseGrowwDate`; fall back to today and set a `statusReason` note that the preview surfaces). Set a `rawFields.notes`-style marker so the commit writes `notes = 'Imported from Groww holdings statement (opening position; avg-cost snapshot)'` — honest about not being real lot history. `isin` = the row's ISIN; `symbol` = trading symbol/name column.

**`src/lib/import/match-instruments.ts`** — `matchInstruments(rows, instruments, savedMappings)`
- Signature: `(rows: ParsedRow[], instruments: Instrument[], savedMappings: Array<{ broker: ImportBroker; brokerSymbol: string; instrumentId: string }>): ParsedRow[]` — pure; the caller (04-04) supplies the instrument universe and saved mappings loaded from the DB.
- Order per row: (1) saved mapping for `(broker, symbol)` → attach `instrumentId`, short-circuit. (2) Groww: exact `isin` match; if multiple instruments share the ISIN across exchanges, auto-pick NSE over BSE (broker-implied), record the pick as overridable. (3) Robinhood: `symbol` match filtered to exchanges ∈ {NASDAQ, NYSE, OTHER} with currency USD; multi-match auto-picks (shown/overridable). (4) No match → `status: 'unmatched'`, add `symbol` to the unmatched set (one entry per unique broker symbol; resolving one fixes all its rows). A row already `unsupported`/`invalid` stays that way (matching doesn't rescue a bad row).

**`src/lib/import/detect-duplicates.ts`** — `detectDuplicates(rows, existingHashes, existingManualTxns, alreadyHeldInstrumentIds)`
- Signature: `(rows: ParsedRow[], existingHashes: Set<string>, existingManualTxns: Array<{ instrumentId: string; type: string; quantity: number; price: number | null; dateISO: string }>, alreadyHeldInstrumentIds: Set<string>): ParsedRow[]` — pure.
- Compute each row's hash (build its `NormalizedRow` and call `computeRowHashes` over the batch so occurrence indices are correct). If the hash ∈ `existingHashes` → `status: 'duplicate'` (`reason: 'already imported'`). Else if it field-matches an existing manual txn (same instrumentId, type, quantity, price, dateISO) → `status: 'duplicate'` (`reason: 'matches a manual entry'`). 
- **Groww already-held rule:** a Groww row whose resolved `instrumentId` ∈ `alreadyHeldInstrumentIds` → `status: 'duplicate'` (`reason: 'instrument already held — snapshot would double-count'`), even with no exact hash/field match (a newer snapshot has different qty/avg). Default handling (skip) is decided by the commit layer's category defaults.
- A row already `unmatched`/`unsupported`/`invalid` is left unchanged.
