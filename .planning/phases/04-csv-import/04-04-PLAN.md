```ts
export async function previewImport(formData: FormData): Promise<ImportPreview>
```

Steps:
1. `const { supabase, accountId } = await requireAuthedContext();` (throws Unauthorized if no session — defense-in-depth alongside RLS).
2. `const file = formData.get('file'); if (!(file instanceof File)) return { ok: false, error: 'No file received' };`
3. `const bytes = new Uint8Array(await file.arrayBuffer());` and `const text = new TextDecoder().decode(bytes);` (text only needed for the Robinhood path).
4. `const fileHash = createHash('sha256').update(bytes).digest('hex');` (`node:crypto`).
5. `const broker = detectBroker(bytes, file.name);` If `'unknown'` → `return { ok: false, error: 'Could not detect broker from file. Expected a Groww .xlsx or a Robinhood activity .csv.' }` (honest failure; the UI offers a manual broker override that re-calls with an explicit broker — accept an optional `formData.get('broker')` override and skip detection when present).
6. Parse: `broker === 'groww' ? parseGroww(bytes) : parseRobinhood(text)`. Wrap in try/catch — an `ImportParseError` becomes `{ ok: false, error: err.message }` (the message already echoes the offending rows).
7. Load the instrument universe and saved mappings for matching (reads, RLS-scoped through the cookie-bound client): all `instruments` (shared reference table — a bounded seed set; select the fields `matchInstruments` needs), and `symbol_mappings` for `(accountId, broker)`. Call `matchInstruments(rows, instruments, savedMappings)`.
8. Load duplicate inputs: the set of existing `import_row_hash` for this account (`select import_row_hash from transactions where account_id = ... and import_row_hash is not null`), the existing MANUAL transactions for field-match (rows with `import_row_hash is null`, mapped to the `{instrumentId,type,quantity,price,dateISO}` shape), and the set of instrument_ids the account already has ANY transaction for (for the Groww already-held rule). Call `detectDuplicates(...)`.
9. Assemble `ImportPreview`: `rows` (as PreviewRow[]), `categories` = counts per RowStatus, `unmatchedSymbols` = unique unmatched broker symbols, `broker`, `fileHash`, `fileName`, `rowCount`. Prior-batch banner: `select imported_count, duplicate_count, created_at from import_batches where account_id = ... and file_hash = ... order by created_at desc limit 1` → if found, set `priorBatch`. Return `{ ok: true, ... }`. **No writes anywhere in this function.**

Preview default categories (04-RESEARCH Pattern 4, resolved): valid→import, duplicate→skip, unsupported→skip (visible), invalid→excluded (only fixable at source), unmatched→excluded until mapped. Encode these as the initial category state the UI reads; the action just reports counts + statuses (the UI owns the toggles).

  <files>src/server-actions/import.ts</files>
  <action>
Add to the same file:

```ts
export async function commitImport(formData: FormData): Promise<ImportResult>
```

Steps:
1. `requireAuthedContext()` (cookie-bound). Read the `file` (same instanceof-File guard) AND a `choices` field: `const choices = JSON.parse(String(formData.get('choices') ?? '{}')) as CommitChoices;`
2. Re-derive `bytes`/`text`/`fileHash`/`broker` and RE-PARSE with the same parser (deterministic → agrees with preview). Never trust client-parsed rows.
3. Resolve mappings from `choices.mappings`: for each `{ brokerSymbol, instrumentId }` use it directly; for each `{ brokerSymbol, create: {...} }` call `const { data: newId, error } = await supabase.rpc('find_or_create_instrument', { p_isin, p_symbol, p_exchange, p_display_name, p_currency });` (the RPC from 04-01 — cookie-bound client, SECURITY DEFINER does the privileged insert). A null/failed result is a hard error returned to the user (never fabricate an id). Build a `brokerSymbol → instrumentId` override map and re-run `matchInstruments` with these as additional saved-style mappings so previously-unmatched rows now resolve.
4. Re-run `detectDuplicates` (same inputs as preview). Decide the final import set per `choices`: always import `valid`; import `duplicate` only if `choices.importDuplicates`; import `unsupported`? no — unsupported types have no ledger representation, they stay skipped/reported (only `importDuplicates` and — if you exposed it — an override are honored). `invalid` and still-`unmatched` rows are NEVER imported. Compute each imported row's `import_row_hash` via `computeRowHashes` over the imported set's NormalizedRows.
5. **Atomic write (partial-commit policy):**
   a. Insert one `import_batches` row (`account_id`, `broker`, `file_name`, `file_hash`, `row_count`, and the computed `imported/skipped/duplicate` counts) → get `batchId`.
   b. Build the transactions payload: for each imported row `{ account_id, instrument_id, transaction_type, quantity, price (null for SPLIT), transaction_date: dateISO, notes (Groww snapshot marker or Robinhood description-derived), import_batch_id: batchId, import_row_hash }`. Do a SINGLE bulk `await supabase.from('transactions').insert(payload)` (one PostgREST statement = atomic; all rows or none). Use plain `.insert`, NOT upsert-ignore — the preview already decided; if the unique index fires it is a real race/logic error the user must see.
   c. On insert error: delete the just-created batch row (`await supabase.from('import_batches').delete().eq('id', batchId)`) as compensation, and `return { ok: false, error }`.
   d. Persist user-resolved mappings to `symbol_mappings` (upsert on the `(account_id, broker, broker_symbol)` unique key) so future imports auto-apply them. A failure here is non-fatal to the import (log/ignore) — the transactions already committed.
6. `revalidatePath('/'); revalidatePath('/holdings');` (imported holdings show up on those pages). Return `{ ok: true, batchId, imported, skipped, duplicates }`.

Note: the post-commit price refresh (so imported holdings show live P&L per success criterion 4) is fired from the client after this returns, reusing the existing `refreshPricesNow` action — do NOT build pricing here (04-RESEARCH Pattern 5). This action's `revalidatePath` is what refreshes the server-rendered pages.
